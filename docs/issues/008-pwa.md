# 008 — PWA: manifest + service worker (app shell only)

Type: AFK

## What to build

Make the site installable so a phone can "Add to Home Screen" and launch fullscreen.

End-to-end behavior:
- Web app manifest (name, icons, theme, `display: standalone`).
- A service worker that caches the **app shell only** (static assets) for installability and
  fast launch. It must **never** cache room state, share links, or secrets. The app still
  requires network for signaling — "offline" means "installed", not "works offline".
- The share-link fragment (the secret) is never seen by the service worker fetch handler
  (fragments aren't sent in requests); confirm nothing in the SW path persists it.

## Acceptance criteria

- [ ] The app passes installability checks and can be added to a phone home screen.
- [ ] Launching from the home screen opens standalone/fullscreen.
- [ ] The service worker caches only static shell assets; no room/secret data is ever cached.
- [ ] A normal transfer still works when launched as an installed PWA.
